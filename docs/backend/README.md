## Interview Readiness Criteria

A topic is considered complete when I can:

* Explain it without referring to notes.
* Describe a real production use case.
* Discuss advantages and disadvantages.
* Explain common failure scenarios.
* Compare it with alternative approaches.
* Draw a basic architecture.
* Answer follow-up questions.
* Relate it to a project from my own experience.
